**`tmux` (Terminal Multiplexer)** 是一种终端复用器。它可以让你：

1. 在一个终端窗口中创建**多个虚拟终端会话（Session）**。
2. 即使 SSH 连接中断、直接关掉终端，**服务器上的 `tmux` 会话及其中运行的程序（如编译、下载、服务）也会继续运行**。
3. 可以随时从任何设备**重新连接（Attach）** 到这个会话。

### 🚀 `tmux` 基础命令和使用方式

| **概念**       | **场景**                                            | **命令 (在 Shell 中输入)**      | **快捷键 (在 tmux 内部操作)** |
| -------------- | --------------------------------------------------- | ------------------------------- | ----------------------------- |
| **创建**       | 开启一个新的会话。                                  | `tmux new -s <会话名>`          | N/A                           |
| **创建并进入** | 如果不指定会话名，`tmux` 会自动创建一个会话并进入。 | `tmux`                          | N/A                           |
| **离开/分离**  | 保持会话在后台运行，安全地离开终端。                | N/A                             | `Ctrl + b` 然后按 `d`         |
| **查看会话**   | 列出当前服务器上所有正在运行的会话。                | `tmux ls` 或 `tmux attach`      | N/A                           |
| **重新连接**   | 重新进入你离开的会话。                              | `tmux attach -t <会话名>`       | N/A                           |
| **关闭/停止**  | 彻底杀死一个或多个会话。                            | `tmux kill-session -t <会话名>` | N/A                           |
| **切换会话**   | 在多个会话之间快速跳转。                            | N/A                             | `Ctrl + b` 然后按 `s`         |

### 💡 进阶：窗格 (Panes) 和窗口 (Windows)

`tmux` 的强大之处在于它可以在一个会话中管理多个窗口和窗格：

| **对象** | **作用**                                                     | **快捷键 (Ctrl + b 后按)**               |
| -------- | ------------------------------------------------------------ | ---------------------------------------- |
| **窗口** | 多个全屏虚拟终端（如桌面环境中的多个桌面）。                 | `c` (Create), `n` (Next), `p` (Previous) |
| **窗格** | 将一个窗口分割成多个子区域（如 `htop` 在左边，代码在右边）。 | `%` (垂直分割), `"` (水平分割)           |
